## Краткое описание работы

В ходе лабораторной работы мы установили .NET SDK, настроили рабочее окружение в VS Code, инициализировали Git-репозиторий и создали первый минимальный веб-сервер на **ASP.NET Core** с использованием шаблона **Minimal API**.

- Горячая перезагрузка (`dotnet watch run`).

## Главные выводы

1. **ASP.NET Core Minimal API** — это максимально простой и быстрый способ создания веб-серверов и API. Всё помещается в один файл `Program.cs`.
2. **Middleware** — мощный инструмент для сквозной обработки запросов (логирование, авторизация, заголовки). Порядок регистрации **очень важен**.
3. **Автоматическая сериализация в JSON** с преобразованием PascalCase → camelCase делает ASP.NET Core совместимым с современными фронтендами (JavaScript/TypeScript).
4. **Hot Reload** (`dotnet watch run`) значительно ускоряет разработку.
5. .NET — кроссплатформенный, высокопроизводительный и удобный фреймворк для backend-разработки.

## Ответы на вопросы

1. **Чем Minimal API отличается от MVC в ASP.NET Core?**  
   Minimal API — это лёгкий подход: всё в `Program.cs`, `MapGet/MapPost`, минимальный overhead. Идеально для микро-сервисов и простых API.  
   **MVC** — полноценный фреймворк с контроллерами, атрибутами `[Route]`, `[HttpGet]`, model binding, Razor Views. Подходит для сложных веб-приложений с большим количеством логики.

2. **Что произойдёт, если поставить `app.MapGet(...)` до `app.Use(...)`?**  
   Middleware, зарегистрированные **после** маршрутов, **не будут выполняться** для этих запросов. Маршруты являются терминальными (short-circuit) — запрос до них не дойдёт. Именно поэтому в документации и в лабораторной рекомендуется **сначала** добавлять все middleware, а потом маршруты.

3. **Почему ASP.NET Core преобразует PascalCase в camelCase при сериализации JSON?**  
   По умолчанию `System.Text.Json` оставляет имена свойств как есть (PascalCase). Однако в ASP.NET Core (Minimal API и MVC) по умолчанию применяется `JsonNamingPolicy.CamelCase`. Это сделано для совместимости с предыдущим поведением Newtonsoft.Json и для соответствия общепринятым стандартам JSON в веб-разработке (JavaScript использует camelCase).

4. **Что означает код ответа HTTP?**  
   - **401 Unauthorized** — клиент не авторизован (нет прав или неверный ключ/токен).  
   - **404 Not Found** — ресурс по указанному URL не найден.  
   - **200 OK** — запрос успешно выполнен, ответ содержит данные.

5. **Чем `dotnet run` отличается от `dotnet watch run`?**  
   `dotnet run` — запускает приложение **один раз**.  
   `dotnet watch run` — включает режим **горячей перезагрузки**: при любом изменении файлов проект автоматически пересобирается и перезапускается (очень удобно при разработке).
